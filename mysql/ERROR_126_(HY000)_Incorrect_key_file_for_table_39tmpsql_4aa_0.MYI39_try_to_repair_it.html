Può capitare quando in /tmp non c'è abbastanza spazio (per esempio ho 500MB liberi in / e sto lavorando con le tabelle degli ordini di 3s, che sono enormi...).

Modificare quindi la variabile tmpdir di my.cnf con un altro path (per esempio /var/tmp, che è su un'altra partizione con più spazio libero, nel mio caso).

Riavviare mysql dopo la modifica al file (/etc/init.d/mysql restart).

 

Note: nel mio caso mysql crea un file temporaneo di ~1G... O.O
