E' capitato che un processo di Samba impazzisse ed andasse al 100% di CPU, questo probabilmente perché continuava a scrivere nei file di log il seguente messaggio

 

[2010/03/26 16:01:17,  0] smbd/notify_inotify.c:inotify_handler(239)
  No data on inotify fd?!

[2010/03/26 16:01:17,  0] smbd/notify_inotify.c:inotify_handler(239)  
No data on inotify fd?!

Per risolvere il problema, aggiungere in /etc/samba/smb.conf la seguente opzione:

kernel change notify = false
