Questa guida va utilizzata in caso si desideri effettuare l'update da un Samba NT4 che si appoggia a openldap a Samba AD.

La procedura consigliata è quella di copiare tutte le configurazioni su una nuova macchina  e ricostruire un samba aggiornato su di essa.

Per prima cosa dobbiamo procedere ad effettuare una copia precisa di ldap dal Server NT4 che chiameremo Server A al Server AD, che chiameremo Server B.

Passi per il backup della configurazione sul Server A:

service slapd stop

slapcat -n 1 -l config.ldif

Bakup del database:

slapcat -n 1 -l db.ldif

 service slapd start

Installazione e importazione sul server B della configurazione ldap

apt-get install libnss-ldap auth-client-config slapd

Stop del servizio ldap:

 service slapd stop

Pulizia della configurazione :

rm -rf /etc/ldap/slapd.d/*

Importazione della configurazione:

slapadd -n 0 -l config.ldif
