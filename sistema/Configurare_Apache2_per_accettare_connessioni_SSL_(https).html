Se non si è già fatto, installare apache2 con il comando

 

aptitude install apache2

 

Su Debian/Ubuntu di default apache è già in ascolto sulla porta 80, quella standard del web. Per fare in modo che sia disponibile anche su https, eseguire questi comandi

 

a2enmod ssl
a2ensite default-ssl
/etc/init.d/apache2 restart

 

Il primo comando abilita il modulo ssl di apache, il secondo abilita il sito ssl, ed il terzo attiva la configurazione.

La posizione predefinita di Debian per i documenti SSL è la stessa del sito web standard, in /var/www. Per modificare questo ed altri parametri aprire il file /etc/apache2/sites-available/default-ssl, in particolare i parametri DocumentRoot e la stanza Directory.

In fase di installazione del pacchetto ssl-cert aptitude creerà dei certificati self-signed automaticamente. Questi certificati non saranno considerati affidabili dai browser, ed anche se consentono il passaggio criptato dei dati non danno alcuna garanzia che si stia comunicando con l'host desiderato.

Per generare dei nuovi certificati autofirmati usare il comando

 

make-ssl-cert generate-default-snakeoil --force-overwrite

 

Per ulteriori configurazioni vedere /usr/share/doc/apache2.2-common/README.Debian.gz.
