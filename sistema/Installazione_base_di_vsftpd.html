vsftpd è un demone sicuro e leggero, veloce da configurare.

apt-get install vsftpd

 

Rimpiazzare la configurazione esistente in /etc/vsftpd.conf con questa:

listen=YES
anonymous_enable=NO
local_enable=YES
write_enable=YES
dirmessage_enable=YES
use_localtime=YES
xferlog_enable=YES
connect_from_port_20=YES
chroot_local_user=YES
secure_chroot_dir=/var/run/vsftpd/empty
pam_service_name=vsftpd
rsa_cert_file=/etc/ssl/private/vsftpd.pem
userlist_deny=YES

 

importanti sono le opzioni anonymous_enable, local_enable, write_enable, userlist_deny. L'ultima consente di definire una lista ristretta di utenti a cui è consentito l'accesso ftp. Il file che contiene questa lista è /etc/vsftpd.user_list.

In caso serva l'accesso al di fuori della rete locale occorre aggiungere alcune opzioni:

pasv_enable=YES
pasv_min_port=6000
pasv_max_port=6025
pasv_address=109.69.131.228
pasv_promiscuous=YES

il parametro pasv_address indica l'ip pubblico, ovviamente sul firewall andranno inoltrate le porte 6000:6025 verso il server ftp.

Per consentire l'accesso agli utenti senz shell modificare il file /etc/pam.d/vsftpd, commentando la riga 

auth   required    pam_shells.so

Per creare un utente con accesso FTP ristretto alla sua sola home directory:

useradd -d /home/ftp/ftp -s /bin/false ftp
passwd ftp 

la shell /bin/false ci assicura che l'utente non avrà i permessi di entrare via ssh. 

Riavviare il demone con il comando

restart vsftpd
