Ecco un elenco dei classici step che è bene considerare ogni volta che si migra un servizio da un server ad un altro.

 

Preparazione nuovo server

  1. installazione del nuovo server
  2. installazione dei programmi necessari con le relative dipendenze

 

Migrazione di prova

ATTENZIONE: nel caso in cui il server sia già in rete con i potenziali utenti (installazione da remoto, solitamente macchine virtuali) è bene assicurarsi che non vi siano accessi accidentali al nuovo sistema (nonostante ip ed hostname siano differenti). Per esempio usare un nome diverso per la directory principale (i.e. accesso web), bloccare le login in modo che possa accedere solo l'amministratore per eventuali verifiche, altre eventuali precauzioni a propria discrezione.

  1. migrazione del filesystem
  2. migrazione del database (se presente)
  3. verifica funzionamento
    • verifica funzionalità
    • verifica permessi sui file

 

Migrazione effettiva

  1. stoppare il servizio sul server in produzione (*)
  2. sincronizzare i dati sul nuovo server (solitamente si effettua una nuova migrazione del database, il filesystem non subisce modifiche)
  3. rendere il servizio disponibile sul nuovo server (nome directory e blocco login precedentemente menzionati)

A volte può capitare di dover assegnare al nuovo server lo stesso ip (ed eventualmente lo stesso hostname, se utilizzato per l'accesso) del vecchio server. In questo caso occorre aggiungere due step:

  1. modificare l'ip del vecchio server (ed eventualmente l'hostname, localmente o sul dns a seconda delle impostazioni della rete) rendendolo quindi inaccessibile agli utenti; i servizi dovrebbero già essere non accessibili perchè stoppati precedentemente (*)
  2. assegnare l'ip al nuovo server (ed eventualmente l'hostname, localmente o sul dns a seconda delle impostazioni di rete)
