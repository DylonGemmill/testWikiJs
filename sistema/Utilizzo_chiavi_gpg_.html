1. Generare una chiave gpg 

https://help.github.com/articles/generating-a-new-gpg-key/

2. Uploadare la chiave appena creata  su un server

gpg --keyserver hkp://pool.sks-keyservers.net --send-key  chiave_appena_creata

3. copiare il file Nextcloud/UfficYO_OC/_support/vim/gnupg.vim   nella cartella .vim/plugin della propria home

4. installare gnupg-agent con: 

sudo apt-get install gnupg-agent

5. aggiungere al file .vimrc nella propria home (se non presente crearlo) la riga

let g:GPGPrimeAgent = 1

6. verificare che nella cartella ~/.gnupg sia presente il file gpg.conf se così non fosse copiare quello in Nextcloud/UfficYO_OC/_support/gnupg/

 7. modificare il file gpg.conf inserendo la propria chiave in default-key e encrypt-to

A questo punto digitando

vim nome_file_criptato.gpg 

verrà chiesto di digitare la password creata al punto 1 insieme alla chiave gpp

Per verificare che un file sia criptato usare il comando file nome_file

Per creare un file criptato:

gpg -e nome_file

e dopo digitare i nomi associati di chi potrà accedere al file (bisogna prima importare chiavi)

Per importare chiavi:

gpg --keyserver keys.gnupg.net --search-keys nome_associato ad esempio

gpg --keyserver keys.gnupg.net --search-keys marco.santacroce

Troubleshooting

Sembrerebbe che nelle vecchie configurazioni il file dove risiedevano le chiavi pubbliche fosse il file pubring.gpg, nelle nuove versioni viene usato il file pubring.kbx.

Per risolvere il problema è possibile importare le chiavi nel nuovo file quindi eseguire:

gpg --import pubring.gpg

Per controllare che le chiavi siano state importate correttamente effettuare :

gpg --list-keys
