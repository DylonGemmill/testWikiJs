Oltre alle classiche regole di IP/subnet è possibile bloccare per nazione. Usando il database pubblico (lite) di MaxMind che associa IP <> nazione si possono creare regole di iptables di questo tipo.

Installare il pacchetto di sistema:

apt install xtables-addons-common

In teoria il pacchetto contiene gli script di download ed aggiornamento delle liste di MaxMind, ma dal gennaio 2019 il sito ha cambiato il formato dei file e non sono ancora stati aggiornate le librerie di iptables con il nuovo. Di fatto i download non funzionano e anche scaricando i file non sono di un formato compatibile. Vedere bdo#918227.

Esiste però uno script di conversione, dal nuovo formato al vecchio compatibile con xtables. Se funziona si può usare (ma su Ubuntu14 non va), altrimenti si può usare un'immagine Docker da far girare una volta alla settimana per effettuare la conversione.

Siccome non sempre è utile avere Docker su un server ho impostato una procedura schedulata (in /etc/cron.d/yetopen) una volta alla settimana su ispc.ufficyo.com che scarica e converte i file di regole usando l'immagine sopra. I file sono disponibili a questo indirizzo, e per aggiornarli su un server basta schedulare un'attività che faccia:

wget -q https://legacy-geoip-csv.ufficyo.com/Legacy-MaxMind-GeoIP-database.tar.gz -O - | tar -xvzf - -C /usr/share/xt_geoip

tutte le domeniche dopo le 4 di mattina.

Creazione regole

Dopo aver aggiornato le liste è possibile creare delle regole con

iptables -A INPUT -m geoip ! --src-cc IT -i eth0 -m conntrack --ctstate NEW -j DROP

con questa regola si blocca tutto il traffico in ingresso su eth0 che non proviene dall'Italia. Si possono specificare più nazioni separate da virgola, ad esempio IT,DE.

Attenzione che inserendo una riga come quella sopra vengono bloccate anche le subnet private, utilizzate per le reti LAN! Prima di quella conviene inserire una riga con:

iptables -I INPUT -i eth0 -s <RETE LAN> -j ACCEPT

in modo da consentire, prima di tutto, che tutto il traffico della rete sia accettato.

È possibile salvare le regole con iptables-save vedere la guida di ubuntu,  oppure fare uno script con le regole e lanciarlo da post-up in /etc/network/interfaces.
