Come si faceva una volta con dischetti / CD ed altri supporti, a volte occorre creare un file compresso suddividendolo in file con una dimensione massima (es limite dei 2Gb di Samba/vFAT).

Tar da solo non è in grado di farlo, si usa il comando split.

Normalmente si può creare un file .tar.gz e poi applicare split. Altrimenti, in un passaggio solo:

tar czf - /etc | split -a 3 -b 1G - nomefiledest.tar.gz

Il parametro a indica il numero di decimali da usare per il conteggio.
Il paremtro b indica la dimensione massima dei singoli file. Deve essere un intero, e si può specificare con i tradizionali suffissi. Per fare 1,5G dovrei scrivere 1500. 

(Per MacOS su chiavetta vFAT: tar czf - work | split -b 4294967295 - /dest/work.tgz.)

Per scompattare:

cat file.tar.gz.* | tar xzvf -
