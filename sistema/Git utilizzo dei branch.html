Spesso usiamo progetti open source di cui modifichiamo i sorgenti del core. In caso di upgrade questo si traduce in problemi tenere traccia delle modifiche e nella loro applicazione. In questo caso viene comoda la funzione branch di Git.

Ad esempio per il progetto Yii:

  1. Creo il progetto yii sul server Git YetOpen
  2. Faccio delle modifiche e ne eseguo il checkout su un nuovo branch
  3. Proseguo lo sviluppo locale sul mio branch

Quando devo aggiornare Yii:

  1. Checkout sul master del progetto yii
  2. Aggiorno i file, push sul server
  3. Merge di master sul branch personalizzato

Tracking di un branch remoto

Creato un branch locale, occorre fare dei passaggi per fare in modo che questo venga pubblicato sul server. Il comando git branch mostra le derivazioni locali, mentre git branch -r mostra quelle presenti sul server.

Se vogliamo tracciare un branch remoto, otteniamone il nome con git branch -r e poi eseguiamo il comando

git branch --track NOMELOCALE NOMEREMOTO

che creerà un branch locale NOMELOCALE sincronizzato con quello remoto NOMEREMOTO.

Nel caso in cui il branch locale già esista:

git branch --set-upstream foo upstream/foo

dove foo è il branch locale e upstream/foo il branch remoto.

 

Push di un branch locale in remoto

Se creo un branch locale e lo voglio in remoto

git push origin branchname

dove origin è il repo remoto e branchname il nome del branch.
