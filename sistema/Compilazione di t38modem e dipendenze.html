Attenzione! t38modem è molto delicato e a quanto pare molto sensibile alle librerie con cui viene compilato. Essendo il sorgente originale abbandonato ci sono stati diversi personaggi che hanno portato avanti le patch per stare in linea con i rilasci di opal.

Una situazione stabile si ha con opal 3.10.3, spandsp 0.6.6 e t38modem dai sorgenti di Henning Holtschneider. Questa configurazione è funzionante!

 

Secondo un messaggio nella ML esiste una patch per compilare t38modem (credo a qs punto) con i sorgenti di Pete, sistemando un bug che probabilmente non è sistemato. Sarebbe da testare.

 

apt-get -y install flex bison libtiff-dev

cd ptlib-2.10.11
export PTLIBDIR=`pwd`
./configure --prefix /opt/t38modem
make
make install


cd spandsp-0.0.6
PKG_CONFIG_PATH=/opt/t38modem/lib/pkgconfig ./configure --prefix /opt/t38modem
make
make install


cd opal-3.10.11
export LIBRARY_PATH=/opt/t38modem/lib/
export CPLUS_INCLUDE_PATH=/opt/t38modem/include/
export C_INCLUDE_PATH=/opt/t38modem/include/
PKG_CONFIG_PATH=/opt/t38modem/lib/pkgconfig ./configure --prefix /opt/t38modem
make
make install

# Sempre dalla dir di Opal
##cd plugins/fax/fax_spandsp

cd t38modem
export OPALDIR=/root/stable/opal-3.10.11
export PTLIBDIR=/root/stable/ptlib-2.10.11 
PKG_CONFIG_PATH=/opt/t38modem/lib/pkgconfig make USE_OPAL=1 USE_UNIX98_PTY=1
cp -v obj_linux_*_opal/t38modem /opt/t38modem/bin

cd opal-3.10.2/plugins/audio/SILK/SILK_SDK
