Invece dell'uso tradizionale di fstab, nelle distribuzioni recenti è viene consigliato AutoFS, un demone in grado di montare automaticamente una directory nel momento in cui viene richiesta. In caso il mount non va a buon fine viene restituito un errore, così da interrompere ad esempio un processo di backup.

 

AutoFS può essere usato per montare una directory di backup, oppure in modo dinamico per montare ad esempio la home directory dell'utente che sta facendo login sul sistema.

 

Per iniziare installare il pacchetto:

apt-get install autofs

 

Il file principale di configurazione è /etc/auto.master, e contiene le definizioni dei percorsi da monitorare. 

AutoFS può essere configurato in modalità diretta o indirietta. Indiretta significa che viene monitorata una directory dentro cui verranno montati i percorsi remoti. Diretta significa configurare uno specifico mountpoint.

Modificare /etc/nsswitch.conf aggiungendo in fondo:

automount: files

 

Montaggio di una directory

Per questa soluzione si usa la modalità diretta. In /etc/auto.master se presente commentare la riga +auto.master ed aggiungere:

/- /etc/auto.backupdati --timeout=60 --ghost

Quindi creare il file /etc/auto.backupdati con:

/backup-dati -fstype=cifs,rw,username=UTENTE,password=PASSWORD,sfu ://IP_SERVER/MOUNTPOINT

riavviare il demone con

service autofs restart

e provare ad accedere al mountpoint (/backup-dati in questo caso), dovrebbe essere montato automaticamente.

Ovviamente per montare un percorso deve essere disponibile il pacchetto relativo. In questo caso smbfs.

 

Nel caso di NFS:

/backup-dati    -soft,intr,bg,rsize=8192,wsize=8192             10.22.22.171:/nfs/backups

 

 

Montaggio delle home degli utenti

Supponendo di avere le home degli utenti condivise via NFS e di voler montare solo le home effettivamente usate sul sistema, in /etc/auto.master aggiungere:

/home   /etc/auto.home

quindi creare /atc/auto.home con:

*   nfs_server:/home/&

usando le wildcard * e &, autofs risolverà automaticamente il percorso da montare e la posizione sul server remoto, così da non dover specificare tutte le condivisioni degli utenti.
