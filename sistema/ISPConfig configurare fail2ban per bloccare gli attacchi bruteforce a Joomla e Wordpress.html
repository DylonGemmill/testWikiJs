Per prevenire gli attacchi bruteforce su Joomla si può usare fail2ban, tenendo monitorato il log degli accessi ed escludendo gli IP che fanno troppi tentativi di GET/POST verso le pagine di login.

Supponendo di aver già installato e configurato fail2ban andiamo a modificare il log di Apache perché sia di un formato utile a fail2ban. Modificare /etc/apache2/apache2.conf e vicino alle altre righe LogFormat aggiungere:

LogFormat "%t %v:%p %h %l %u \"%r\" %>s %O \"%{Referer}i\" \"%{User-Agent}i\"" fail2ban

Modificare poi /etc/apache2/conf.d/other-vhosts-access-log e sulla la riga CustomLog modificare vhost_combined con fail2ban. Riavviare apache.

Creare un file /etc/fail2ban/filter.d/apache-wordpress.conf con questo contenuto:

[Definition]
failregex = <HOST> - - .*(POST|GET) .*/wp-login.php HTTP.*
ignoreregex =

creare anche /etc/fail2ban/filter.d/apache-joomla.conf con questo contenuto:

[Definition] 
failregex = .* <HOST> - - .*(POST|GET) .*administrator/index.php HTTP.*
ignoreregex =

modificare /etc/fail2ban/jail.local ed aggiungere:

[apache-wordpress]
enabled = true
banaction = iptables-allports
bantime = 7200
port = all 
filter = apache-wordpress
logpath = /var/log/apache2/other_vhosts_access.log
maxretry = 5 

[apache-joomla]
enabled = true
banaction = iptables-allports
bantime = 7200
port = all 
filter = apache-joomla
logpath = /var/log/apache2/other_vhosts_access.log
maxretry = 5

 

Riavviare fail2ban.

Attenzione che il filtro potrebbe impiegarci parecchio tempo ad attivarsi. Soprattutto se il file other_vhosts_access è molto grande, fail2ban inizierà a bannare gli attacchi dopo alcune decine di minuti.
