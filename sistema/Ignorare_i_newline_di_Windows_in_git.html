Per ignorare i newline di Windows, che solitamente compaiono come ^M in qualche editor, utilizzare la seguente opzione nei comandi git:

-Xignore-space-at-eol

testata con cherry-pick ma dovrebbe funzionare anche con merge, diff e altri...

 

fonte: http://stackoverflow.com/questions/861995/is-it-possible-for-git-merge-to-ignore-line-ending-differences

 

 

E' anche presente un'opzione per gestire proprio i caratteri di fine riga, per una gestione automatica digitare

git config --global core.autocrlf input

fonte: https://help.github.com/articles/dealing-with-line-endings/
