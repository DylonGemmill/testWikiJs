Installare il pacchetto fetchmail modificare o creare /etc/fetchmailrc:

# Opzioni globali
set daemon 120
set syslog
set no bouncemail
set postmaster support@yetopen.it

# Opzioni server da applicare a tutti i server
defaults bad-header accept proto pop3 timeout 60 uidl antispam -1 options ssl 

poll pop3s.aruba.it
    user esempio@yetopen.it pass "password" to esempio@yetopen.it keep

poll pop3s.aruba.it envelope 1 "Delivered-To" localdomains yetopen.it:   
   user catchall@yetopen.it with pass "password" to * here

 

I parametri principali:

  • daemon 120: intervallo di controllo della posta, in secondi;
  • bad-header accept: accetta le mail con intestazioni non corrette (capita con aruba o altri);
  • timeout 60: timeout di connessione del pop3;
  • options ssl: connessione SSL al pop3 (di default su aruba).

Le opzioni definite nel defaults possono essere specificate (o modificate) in ogni istanza di definizione dei server (poll).

La sezione poll identifica il polling di un server, per ogni server (a parità di impostazioni) posso definire più utenti. Nel caso ci siano diversi utenti ed una catchall occorre definire due poll.

Nel caso della catchall il numero tra envelope e Delivered-To indica il numero di righe Delivered-To da ignorare. La riga in questione serve a fetchmail per identificare il destinatario originale della mail, e ad esempio nel caso di Arubaci sono due righe di quell'intestazione e la prima deve essere ignorata. In altri ambienti non dovrebbe servire.
