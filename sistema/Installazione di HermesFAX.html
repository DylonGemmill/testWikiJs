Scaricare HermesFax dal sito di OpenIT e scompattare il contenuto in /var/www (crea automaticamente la sottodir hermesfax).

 

Installare le dipendenze con il seguente comando:

apt-get install libapache2-mod-auth-mysql libapache2-mod-perl2 libdbi-perl libdbd-mysql-perl libgd2-xpm libgd2-xpm-dev libgd-tools libcrack2 libcrack2-dev crack-common cracklib-runtime libtiff4 libtiff4-dev  netpbm  libgd-gd2-perl libmime-lite-perl libio-socket-ssl-perl libhtml-format-perl htmldoc psutils pstotext texlive-latex-base texlive-latex-extra hylafax-server cups cups-bsd
cpan -fi Crypt::Cracklib
ln -sf /etc/fonts/conf.avail/10-autohint.conf /etc/fonts/conf.d/
a2enmod auth_mysql

Per l'installazione di Cracklib alle richieste rispondere con i predefiniti.

Occorre ora compilare tif22pnm:

wget -q ftp://ubuntu.tiscali.nl/pub/mirror/gentoo/distfiles/tif22pnm-0.14.tar.gz -O - | tar xvzf - -C /usr/src
cd /usr/src/tif22*
./configure 
./do.sh compile
cp tif22pnm png22pnm /usr/local/bin/

 

 

Posizionarsi in /var/www/hermes e lanciare 

./runme

Verranno poste una serie di domande, rispondere come segue:

  • installazione pacchetti debian: NO;
  • usare le GD: SI;
  • installazione di TIF22PNM: NO;
  • path di tif22pnm: /usr/local/bin/tif22pnm;
  • hylafax versione 4.2.x: no;
  • utente  hylafax: uucp;
  • root di hylafax: /var/spool/hylafax/;
  • uso di ssl: no;
  • domain extension: dominio email della rete;
  • superuser: admin;
  • password: (password);

Modificare in /etc/hermes/hermes.conf il parametro RootNet con la rete di amministrazione (indispensabile per configurare utenti e dispositivi), a cui sarà consentito modificare la configurazione.

Andare poi sul web di installazione e cliccare sul logo HermesFax in alto a sinistra per iniziare la configurazione.

Creare un utente master e aggiungere i dispositivi fax.
