Installazione akeeba sul sito 

Sito di akeeba: http://www.akeebabackup.com
:: Spostamento sito con Akeeba http://www.akeebabackup.com (ex joomlapack) ::
Scaricare (http://www.akeebabackup.com/download/akeeba-backup-core-for-joomla/index.html) e installare il plugin akeeba sul sito da migrare.
Una volta installato, dal pannello di amministrazione, selezionare
Componenti > Akeeba Backup > Configuration
Alla voce "Archiver engine" selezionare "ZIP format".
Selezionare quindi la voce di menu "Backup Now", inserire una descrizione per il backup ed avviare la procedura cliccando su "Backup Now".
Una volta terminato il backup, andare nella sezione "Administer Backup Files" e scaricare il backup appena creato (assicurarsi che sia un archivio zip).
A questo punto, occorre uploadare l'archivio sul nuovo spazio web e decomprimerlo.
Se abbiamo un accesso ssh, è possibile decomprimere l'archivio con il comando unzip. Visitando poi col browser la directory di destinazione, partirà l'installazione della copia di backup.
Se non abbiamo un accesso ssh, è possibile decomprimere l'archivio utilizzando uno script chiamato Kickstart, scaricabile dal sito di Akeeba (http://www.akeebabackup.com/download/akeeba-kickstart/index.html). Scarichiamo l'archivio ed estraiamo il file kickstart.php, uploadiamo quindi il file nella stessa directory dell'archivio. Assicuriamoci che sia i file che la cartella siano scrivibili da apache. Visitiamo quindi la pagina http://nuovosito.it/kickstart.php per selezionare le opzioni necessarie e dare il via alla decompressione dell'archivio. Alla fine dell'operazione, lo script dovrebbe portare direttamente all'installazione della copia di backup, in alternativa visitare la cartella dal browser.
Se nemmeno questo metodo dovesse funzionare, si può sempre decomprimere l'archivio in una cartella locale ed uploadare i file decompressi.
Il primo passo dell'installazione (come per l'installazione di joomla) si occupa di verificare i permessi sulle cartelle e le impostazioni di apache. Modificare i permessi come suggerito dall'installer (chown -R apache:apache *).
Viene anche suggerito di disabilitare il safe_mode di php; se si dispone di un pannello plesk, entrare nel pannello del dominio relativo e selezionare "Web Hosting Settings". Alla voce "PHP support" togliere il check al "PHP safe_mode".

Ripristino del backup di Akeeba

Caricare l'archivio del backup sul nuovo spazio web e decomprimerlo.

Se abbiamo un accesso ssh, è possibile decomprimere l'archivio con il comando unzip. Visitando poi col browser la directory di destinazione, partirà l'installazione della copia di backup.

Decompressione backup con Kickstart

Se non abbiamo un accesso ssh, è possibile decomprimere l'archivio utilizzando uno script chiamato Kickstart, scaricabile dal sito di Akeeba (http://www.akeebabackup.com/download/akeeba-kickstart/index.html).

Scarichiamo l'archivio ed estraiamo il file kickstart.php, uploadiamo quindi il file nella stessa directory dell'archivio. Assicuriamoci che sia i file che la cartella siano scrivibili da apache. Visitiamo quindi la pagina http://nuovosito.it/kickstart.php e seguiamo le istruzioni a schermo, se i file di backup sono con estensione .jsp verrà richiesta una password dell'archivio che si può trovare in Bitwarden. Alla fine dell'operazione, lo script dovrebbe portare direttamente all'installazione della copia di backup, in alternativa visitare la cartella dal browser.

In ambiente di sviluppo ricordarsi di disabilitare l'invio delle mail durante l'installazione. Per farlo togliamo la spunta a "Turn on mail sending" nella sezione "Site Parameters" nel passo di installazione "Site Setup".

Dopo il termine della procedura guidata, in ambiente cambiamo i seguenti parametri in configuration.php:

  • $cache_handler = '';
  • $force_ssl = '0';
  • $live_site = '';

 

Se nemmeno questo metodo dovesse funzionare, si può sempre decomprimere l'archivio in una cartella locale ed uploadare i file decompressi. Per far partire l'installazione occorre visitare la directory di destinazione da browser.
