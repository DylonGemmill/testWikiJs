Un'operazione che potrebbe essere necessaria quando si effettua il restyling di un sito web joomla è quella di trasferire gli articoli e gli utenti dal vecchio al nuovo sito Requisito fondamentale è che i due siti siano stati realizzati con la stessa versione di Joomla (es. 1.5.x).


Quanto sotto descritto presupppone che sappiate usare la web application phpMyAdmin e abbiate nozione di database MySQL. Non mi assumo nessuna responsabilità in caso di perdita di dati a seguito dell'errata esecuzione di operazioni sul database.

Esportazione delle tabelle

Accedere al database del sito di origine utilizzando la web application phpMyAdmin che la maggior parte degli hosting provider rendono disponibile ai propri clienti.

Nella porzione sinistra della pagina dovrebbe essere presente l'elenco delle tabelle del database, se ciò non fosse dovrete selezionare (sempre nella parte sinistra della pagina) il database da cui esportare i dati.

Una volta selezionato il database cliccate sul link Esporta (in alto a destra) per passare alla pagina di esportazione dei dati.
L'esportazione di un database MySQL consiste nel creare un file di testo contenente sia le istruzioni per creare le tabelle stesse su un altro database sia quelle per riempirle con i dati.

Nella pagina di esportazione, sezione Esporta, selezionate solo le tabelle che volete esportare. Per farlo tenete premuto il pulsante CRTL sulla tastiera e scorrete la lista delle tabelle cliccando su quelle che vi interessano.

Per i contenuti le tabelle da esportare sono le seguenti:

jos_categories

jos_sections

jos_content

jos_content_frontpage (opzionale)

jos_content_rating (opzionale)

La tabelle jos_content_frontpage contiene gli id degli articoli pubblicati in homepage mentre la tabella jos_content_rating contiene i voti sugli articoli dati dai visitatori. Se non vi interessano i dati contenuti in queste tabelle potete anche ignorarle

Per la migrazione degli utenti le tabelle sono le seguenti:

jos_users

jos_core_acl_aro

jos_core_acl_aro_groups

jos_core_acl_aro_map

jos_core_acl_aro_sections

jos_core_acl_groups_aro_map

 

Se non desiderate trasferire gli utenti non sarà necessario esportare le tabelle contenenti gli utenti (sembra scontato ma...)Non è necessario esportare quelle tabelle che dovessero risultare vuote (è una ovvietà ma... ;D)

Nella sezione Struttura selezionate le opzioni "Aggiungi DROP TABLE" e "Aggiungi IF NOT EXISTS", in questo modo, quando effettuerete l'importazione dei dati nel database di destinazione, le tabelle già presenti verranno cancellate, ricreate e riempite con i contenuti originali.

Assicuratevi che nella sezione Dati siano spuntate le opzioni per l'inserimento completo ed esteso dei dati, spuntate la casella Salva con nome... e cliccate sul pulsante Esegui; vi verrò chiesto di salvare un file che porta il nome del database di origine ed estensione .sql

Importazione delle tabelle

A questo punto accedete al database di destinazione usando sempre la web application phpMyAdmin, selezionate il database in cui importare i dati, cliccate sul link Importa (in alto a destra nella pagina), selezionate il file .sql esportato in precedenza e cliccate sul pulsante Esegui.

Attenzione! importando la tabella jos_users si perderanno gli l'account utente già presenti che verranno sostituiti da quelli del sito di origine.
