PHP ha già al suo interno i driver di connessione ad un server SQL Server, qundi se si conosce già hostname e porta si può installare il pacchetto php5-sybase.

Non sempre però questo è sufficiente, in alcuni casi (ad esempio nelle versioni recenti del server) MSSQL è configurato in modalità Dynamic Port Allocation, il che significa che ad ogni avvio del servizio verrà usata una porta potenzialmente diversa. Per connettersi in questa modalità è indispensabile passare attraverso ODBC.

 

apt-get install php5-sybase unixodbc tdsodbc

 

Aprire il file /etc/odbcinst.ini e scrivere le specifiche del driver:

[TDS]
Description = FreeTDS Driver
Driver = /usr/lib/x86_64-linux-gnu/odbc/libtdsodbc.so
Setup = /usr/lib/x86_64-linux-gnu/odbc/libtdsS.so

e poi in /etc/odbc.ini i dettagli di connessione: 

[SQLSRV01]
Description = SQL Server test
Driver = TDS
Trace = No
Server = IP_DEL_SERVER\NOME_ISTANZA
TDS_Version = 9.0
#Database = VGISystemVend001
#ReadOnly = Yes

La sintassi IP_DEL_SERVER\NOME_ISTANZA è indispensabile per la modalità con porta dinamica (in alternativa mettere il solo IP e una nuova riga di config con Port=xxx funzionerebbe comunque).

La versione TDS di solito va bene, si può provare con la versione 8 se ci sono problemi. Se entrambe non funzionano vedere qui sotto per avere informazioni sull'istanza.

La riga Driver fa riferimento alla stanza che abbiamo messo in odbcinst.ini. 

 

Scritte le configurazioni dobbiamo fare in modo che ODBC le installi con i comandi seguenti. Da notare che vanno eseguiti ogni volta che vengono modificati i rispettivi file! 

odbcinst -i -s -f /etc/odbc.ini

 

A questo punto possiamo fare un test di connessione con il comando:

isql -v SQLSRV01 nome_utente password

dove SQLSRV01 è il nome dell'istanza che abbiamo specificato in odbc.ini. Se tutto ok in php possiamo stabilire una connessione con queste istruzioni:

$db = new PDO("dblib:host=SQLSRV01;dbname=NOMEDB","USERNAME","PASSWORD");

 

Per PHP 5.5 occorre il pacchetto php5-odbc e la connessione deve essere modificata in questo modo:

$db = new PDO("odbc:dsn=SQLSRV01;dbname=NOMEDB", "USERNAME","PASSWORD");

 Attenzione: PHP utilizza le istanze configurate in freetds.conf e non quelle di odbc.ini.

 

Troubleshooting

Installare il pacchetto freetds-bin ed eseguire:

tsql -H host_o_ip -L

Dovrebbe visualizzare alcune informazioni sul server SQL, ad esempio:

     ServerName IT007410S00555
   InstanceName VGISYSTEM
    IsClustered No
        Version 9.00.5000.00
            tcp 1223
             np \\IT007410S00555\pipe\MSSQL$VGISYSTEM\sql\query

 

Per un test di connessione eseguire (la porta è quella visualizzata con il comando precedente)

tsql -S server -p porta -U utente -P password
