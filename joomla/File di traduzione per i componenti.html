Ogni estensione di Joomla (sia essa plugin, tema, modulo, ecc...) può avere dei file lingua associati. In Joomla questi file si trovano nei path language e administrator/language (rispettivamente i file di traduzione per front end e back end). In questi due percorsi si trovano quindi diverse cartelle, ognuna con la sigla della lingua interessata (it-IT, en-GB, ecc... le prime due lettere sono in codice ISO 639 minuscolo mentre le ultime due in ISO 3166 maiuscolo, ma sempre della stessa lingua). All'interno di queste cartelle ci sono infine i file con le traduzioni. Ogni file ha nome composto come segue:

it-IT.tpl_h2raise_template.ini

dove it-IT è il codice della lingua, tpl_ identifica il tipo di estensione (in questo caso un template) e h2raise_template è il nome dell'estensione.

La sintassi del file è del tipo:

ESTENSIONE_DESCRIZIONE=Descrizione a video

Il valore chiave non deve contenere spazi e dev'essere univoco in tutto Joomla, per cui si antepone sempre il nome dell'estensione, esempio:

TPL_H2RAISE_DESCRIZIONE=Descrizione del tema h2raise

Nel pacchetto di installazione dell'estensione, i file di traduzione possono essere messi dove si vuole: l'importante è referenziarli correttamente nel file XML di installazione. Per esempio nel caso di un template, avremo la sezione relativa alla lingua nel file templateDetails.xml, che sarà di questo tipo:

<languages folder="language">
  <language tag="en-GB">en-GB/en-GB.tpl_h2raise_template.ini</language>
  <language tag="it-IT">it-IT/it-IT.tpl_h2raise_template.ini</language>
</languages>

La sezione languages ci dice che i file di traduzione sono presenti nell'archivio secondo questo schema:

language/
language/it-IT/
language/it-IT/it-IT.tpl_h2raise_template.ini
language/en-GB/
language/en-GB/en-GB.tpl_h2raise_template.ini

Sarà quindi poi possibile richiamare le stringhe di traduzione all'interno delle pagine PHP dell'estensione in questo modo:

echo JText::_('TPL_H2RAISE_DESCRIZIONE');
