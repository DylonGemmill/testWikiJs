Ambiente di sviluppo

Inizialmente, per avere un buon ambiente di sviluppo, bisognava scaricare l'SDK di android dal sito ufficiale, installare Eclipse e configurarlo con dei plugin aggiuntivi per utilizzare l'SDK desiderato. L'SDK aveva un eseguibile di nome android che apriva una finestra dalla quale era possibile selezionare ed installare le varie parti dell'SDK. Per aggiungere le parti necessarie ad Eclipse occorreva andare nella sua gestione dei plugin ed aggiungere un repository dal quale installare il software per la compatibilità con android.

 

Ora scaricando dal sito l'SDK, è presente anche un eclipse standalone già configurato, decisamente più comodo.

 

Google mette anche a disposizione un pacchetto di nome Android Studio, composto dall'SDK e da un IDE basato su IntelliJ IDEA customizzato apposta per l'occasione. Un avviso sul sito spiega che l'IDE non è ancora stabile (beta) quindi per evitare rogne consigliano di usare ancora Eclipse.

 

Aggiungere una libreria esterna ad Eclipse

A volte può capitare di dover aggiungere una libreria esterna al nostro progetto android. Per prima cosa occorre scaricare il pacchetto JAR necessario. Successivamente occorre andare nella directory lib del nostro progetto, click destro, Import > General > File System, quindi Browse e selezionare la directory dove si trova il file JAR precedentemente scaricato. Eclipse mostrerà la lista dei contenuti nella cartella, selezionare il nostro JAR e cliccare su Finish.

Nel nostro caso è stata usata ksoap2 (https://code.google.com/p/ksoap2-android/) per effettuare chiamate SOAP verso Feng Office.
