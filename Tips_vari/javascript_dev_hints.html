Quando si lavora con chiamate SOAP, o altri webservices, è possibile che, cercando la variabile con la risposta, si utilizzi l'attributo data in questo modo

zres.xml.getElementsByTagName("return")[0].firstChild.data

Può però capitare di ottenere una risposta troncata (tipicamente a 4096 caratteri). Questo perché alcuni browser (tra cui Firefox) spezzano le stringhe troppo lunghe in più nodi. Per accedere all'intera risposta bisogna utilizzare l'attributo textContent invece di accedere al firstChild, in questo modo

zres.xml.getElementsByTagName("return")[0].textContent

fonte: http://stackoverflow.com/questions/1967741/is-there-a-4096-character-limit-for-javascript-xml-text-nodes

 

A volte si pensa di copiare un oggetto con un semplice assegnamento, in realtà non viene copiato ma ci si rifeirsce sempre allo stesso elemento.

Esempio:

a = [1, 2, 3];
b = a;
b.push (5);
console.log (a); // [1, 2, 3, 5]
console.log (b); // [1, 2, 3, 5]
