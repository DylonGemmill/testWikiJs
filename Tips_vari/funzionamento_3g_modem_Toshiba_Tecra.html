Il modem è supportato da ubuntu ma non viene attivato all'avvio, quindi da networkmanager non è possibile abilitarlo e disabilitarlo.

Il modem è collegato come dispositivo usb, ecco alcuni dati presi da lsusb -v per poterlo riconoscere:
Bus 001 Device 002: ID 0930:130b Toshiba Corp.
iManufacturer           1 Ericsson
iInterface             14 Toshiba F3507g Mobile Broadband Network Adapter

Per poterlo abilitare è necessario installare:
aptitude install fnfxd
aptitude install toshutils
aptitude install thinkfinger-tools

Ora dovrebbe essere disponibile il comando toshset che permette, tra le altre cose, di abilitare/disabilitare il modem 3g.

Comandi utili:
toshset -3g status
toshset -3g on
toshset -3g off

Una soluzione può essere quella di aggiungerlo alla combinazione di tasti fn necessaria per abilitare il bluetooth. Tale combinazione esegue lo script /etc/acpi/tosh-wireless.sh che, con le aggiunte del 3g, diventa:

#!/bin/sh

test -f /usr/share/acpi-support/key-constants || exit 0

. /usr/share/acpi-support/state-funcs

if isAnyWirelessPoweredOn; then
    if [ -x /usr/bin/toshset ]; then
        if `toshset -bluetooth | grep -q attached`; then
                toshset -3g off
                toshset -bluetooth off
                toggleAllWirelessStates
        else
                toshset -bluetooth on
                toshset -3g on
        fi
    else
        toggleAllWirelessStates
    fi
else
        toggleAllWirelessStates
fi

Sui Toshiba Tecra di G&V è stata anche aggiunta l'opzione anche in /etc/rc.local, come segue:
toshset -3g on
in modo che il modulo umts sia attivo subito dopo l'avvio del pc.

ATTENZIONE: non confondere l'abilitazione toshset con l'abilitazione da networkmanager. Quando il dispositivo è abilitato da toshset possiamo scegliere se abilitare/disabilitare la rete mobile da networkmanager. Quando il dispositivo è disabilitato da toshset, pur settando da networkmanager non succede niente.
