Yii include le widget di JQuery UI. Tali widget sono caratterizzati da delle proprietà (options) e degli eventi (events) ai quali possono essere assegnate delle azioni. Mentre in JQuery puro questi 2 set di opzioni vengono impostati in due array differenti, in Yii vengono inseriti tutti nell'array 'options'.

Bisogna poi fare attenzione alla dichiarazione degli events, perchè bisogna rispettare la convenzione di Yii (js:) e non è possibile chiamare una funzione direttamente.

Esempio:

 

// stampo la lista ordinabile
$this->widget ('zii.widgets.jui.CJuiSelectable', array (
    'items' => $sortable_elements,
    // opzioni del componente jui (http://jqueryui.com/demos/sortable/)
    'options' => array (
        'opacity' => 0.6,
        'selecting' => 'js: function (event, ui) {process.resetCompatible ();}',
        'unselecting' => 'js: function (event, ui) {process.resetCompatible ();}',
    ),
    'htmlOptions' => array (
        'title' => 'Clicca per selezionare.',
        //'style' => 'cursor: pointer;',
        'class' => 'components'
    ),
));

 

 

In questo caso gli eventi selecting e unselecting sono nell'array options. Da notare che non è possibile chiamare direttamente la funzione, una dichiarazione del tipo:

 

    'options' => array (
        'opacity' => 0.6,
        'selecting' => 'js: process.resetCompatible ();',
        'unselecting' => 'js: process.resetCompatible ();',
    ),

 

non funziona...
