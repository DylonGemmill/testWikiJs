Per mitigare le problematiche avute con Nextcloud abbiamo introdotto le seguenti policy su AD:

  • Richiesta di conferma cancellazione file anche per i file locali
    Configurazione utente > Criteri > Modelli amministrativi (...) > Componenti di Windows > Esplora file: Visualizza finestra di conferma per la cancellazione file: Attivo
    https://www.isumsoft.com/it/enable-delete-confirmation-dialog-box-in-windows-10-8/#way2
  • Aumentati i pixel di trascinamento dei file, per cercare di ridurre gli spostamenti indesiderati di una cartella nell'altra. Non c'è una proprietà del GPO ma bisogna modificare il registro di Windows
    Configurazione utente > Preferenze > Impostazioni di Windows > Registro di sistema
    DragHeight: 50
    DragWidth: 50
    https://winaero.com/blog/change-drag-drop-sensitivity-windows-10/
  • Non sincronizzare i file di lock di Autocad (.dwl e .dwl2). Le esclusioni di Nextcloud sono nel file  %appdata%\Nextcloud\sync-exclude.lst, per fare in modo di averlo uguale su tutti i PC abbiamo un file master in \\fileserver\pubblica\nextcloud\sync-exclude.lst che viene copiato tramite GPO
    Configurazione utente > Impostazioni di Windows > File
    Quando si definisce la policy è importante che il file sorgente sia accessibile dal gruppo domain users e domain computers e che venga specificato anche il nome file di destinazione.
