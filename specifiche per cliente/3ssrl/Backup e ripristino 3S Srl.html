Backup singole caselle

E' un backup che non impegna eccessivamente il server (dipende dalla dimensione delle caselle ovviamente), esporta un file compresso con la posta e le impostazioni dell'account. Facciamo un backup completo una volta alla settimana (sabato ore 21:22) e due backup incrementali al giorno. Valuta tu se vuoi togliere il salvataggio di metà giornata, noi lo facciamo per i nostri clienti visto che non occupa molto spazio e non rallenta, ma poi dipende dall'uso che faranno del server. Vengono conservati 20gg di salvataggi.
I file sono memorizzati in /dati/backup/zmbkpose.

Per il ripristino di una casella:
zmbkpose -r -a nomecasella@dominio.it
ATTENZIONE che se non viene specificato il parametro "-a" ripristina TUTTE le caselle!

 

Backup completo

Backup di tutta la /opt/zimbra da usare in caso di disaster recovery. Da qui è possibile solo estrarre l'intero Zimbra, non si possono fare ripristini puntuali. Viene eseguito tutte le domeniche mattina alle 3, viene fatto un rsync a caldo ed uno più veloce con Zimbra fermo, quindi comporta un breve stop (pochi minuti) dei servizi Zimbra.
Vengono conservati i backup degli ultimi 30gg.
I file sono memorizzati in /dati/backup/dr/. C'è anche una directory /dati/backup/tmp/ che viene usata per l'rsync, non includerla nei backup di rsnapshot!

Per ripristinare da zero:

  • installare il sistema operativo
  • installare zimbra "pulito" (non serve fare una configurazione dettagliata visto che tanto dovrà essere ripristinato dal backup)
  • fermare tutti i servizi Zimbra
  • spostare la /opt/zimbra in /opt/zimbra_TMP
  • mkdir /opt/zimbra ; cd /opt/zimbra
  • dar -w -x /dati/backup/dr/

Una nota: deve essere la parte iniziale del file. Ad esempio se il file di backup si chiama 06_ZimbraBackup_20140210_FULL.1.dar il comando deve essere lanciato con 06_ZimbraBackup_20140210_FULL.
Fatto questo si possono ripristinare tutte le caselle più recenti con Zmbkpose.
