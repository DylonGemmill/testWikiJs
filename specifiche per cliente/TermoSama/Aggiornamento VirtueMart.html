Per aggiornare VirtueMart bisogna considerare, oltre alle patch di YetOpen, anche alle limitazioni di upload del server.

Si consiglia quindi di caricare i pacchetti da installare nell'apposita cartella tmp ed effettuare l'installazione da lì.

Trasferimento pacchetti

$ scp Joomla_2.5.x_to_2.5.9-Stable-Patch_Package.zip com_virtuemart.2.0.18a_extract_first.targz.zip it_it_virtuemart_language_pack.zip it-IT_joomla_lang_full_2.5.9v1.zip sss:/tmp

estrazione (sul server)

# cd ~/web
# unzip /tmp/Joomla_2.5.x_to_2.5.9-Stable-Patch_Package.zip -d tmp/Joomla_2.5.x_to_2.5.9-Stable-Patch_Package
# unzip /tmp/it-IT_joomla_lang_full_2.5.9v1.zip -d tmp/it-IT_joomla_lang_full_2.5.9v1
# unzip /tmp/com_virtuemart.2.0.18a_extract_first.targz.zip -d /tmp/com_virtuemart.2.0.18a_extract_first
# mkdir tmp/com_virtuemart.2.0.18a
# tar xf /tmp/com_virtuemart.2.0.18a_extract_first/com_virtuemart.2.0.18a.tar.gz -C tmp/com_virtuemart.2.0.18a
# mkdir tmp/com_virtuemart.2.0.18a_ext_aio
# tar xf /tmp/com_virtuemart.2.0.18a_extract_first/com_virtuemart.2.0.18a_ext_aio.tar.gz -C tmp/com_virtuemart.2.0.18a_ext_aio/
# unzip /tmp/it_it_virtuemart_language_pack.zip -d tmp/it_it_virtuemart_language_pack

attribuire i permessi necessari alla directory

# chown -R supersamastore:supersamastore tmp

Dalla console di amministrazione di Joomla, andare in Estensioni > Gestione estensioni eseguire l'installazione dalle seguenti cartelle

/var/www/supersamastore.it/htdocs/tmp/Joomla_2.5.x_to_2.5.9-Stable-Patch_Package
/var/www/supersamastore.it/htdocs/tmp/it-IT_joomla_lang_full_2.5.9v1
/var/www/supersamastore.it/htdocs/tmp/com_virtuemart.2.0.18a
/var/www/supersamastore.it/htdocs/tmp/com_virtuemart.2.0.18a_ext_aio
/var/www/supersamastore.it/htdocs/tmp/it_it_virtuemart_language_pack

Aggiornare quindi le tabelle dei plugin andando in Componenti > virtuemart-aio per mezzo del pulsante Aggiornamento tabelle plug-in VM

Verificare che tutte le voci di menu che portano alla home page di VM abbiano &view=virtuemart, altrimenti compare sempre un warning
