Il nostro phpMyFaq non ci permette di creare i pdf di articoli con le immagini perché cerca di ricavarle dall'indirizzo http corrispondente. Il server quindi si troverà davanti alla login di Shibboleth invece dell'immagine vera e propria e TCPDF darà un errore. In allegato una patch per TCPDF che modifica il path delle immagini in modo che vengano prese localmente, senza passare da http (e quindi da Shibboleth).

Durante la scrittura dell'articolo, inoltre, i path delle immagini selezionate sono sbagliati. L'errore è dovuto al symlink /var/www > /home/www. Le operazioni che effettua pmf per calcolare gli url, sono infatti:

- prendere il vero path dell'immagine (/home/www/kb/images/...)

- ricavare la wwwroot con apache (/var/www)

- togliere la wwwroot dal path: l'operazione fallisce perchè non trova /var/www all'inizio del path

- aggiungere l'hostname

di conseguenza il path risulta una cosa del tipo http://www.mioserver.com/home/www/kb/images... che è errato.

La seconda patch modifica il wwwroot con /home/www, in modo che la sostituzione avvenga correttamente.
