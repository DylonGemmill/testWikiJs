Sintomi

All'avvio, dopo la schermata del produttore del PC, rimane lo schermo nero con la sola possibilità di muovere il cursore del mouse.

Non è possibile accedere nè alla modalità provvisoria nè al ripristino del sistema.

Anche smontanto la batteria e lasciando il PC senza corrente per diverso tempo, il comportamento non cambia.

Anche tenendo premuto il pulsante di accensione finc'è tutte le spie del PC si spengono e riavviando, il comportamento non cambia.

 

Soluzione

Probabilmente i file per il ripristino dell'ibernazione sono stati corrotti ed il sistema non riesce a partire. E' possibile rimuovere tali file per far eseguire a Windows un avvio completo: I DOCUMENTI CHE ERANO APERTI PRIMA DELL'IBERNAZIONE POTREBBERO ESSERE PERSI O DANNEGGIATI

Avviare il PC con una live Linux che abbia i software per gestire i filesystem NTFS (ntfsprogs o altri a seconda della distro, noi abbiamo usato Knoppix 7.4 e ha il software necessario).

Montare la partizione di Windows: se si usa il file manager dovrebbe comparire un messaggio di avviso che è presente il file di ibernazione. Tra le opzioni possibili selezionare quella di continuare, anche se occorre cancellare il file. Una volta montato il disco rinominare o cancellare i file hiberfil.sys e pagefile.sys, se presenti.

Smontare il disco e riavviare.

 

Troubleshooting

Non riesco a montare comunque il disco a causa di un errore di NTFS.

Eseguire un controllo del filesystem con i comandi:

 

ntfsck /dev/sda4
ntfsfix /dev/sda4

 

dove /dev/sda4 è la partizione di Windows.
