Per evitare di avere un file enorme ibdata1 enorme, è possibile aggiungere la riga

innodb_file_per_table

nel file my.cnf nella sezione [mysqld]. In questo modo mysql creerà un file apposito per ogni tabella.

Può capitare però, che su sistemi con un grosso numero di database e/o tabelle, mysql non parta con l'errore

ERROR 1018 (HY000): Can't read dir of '.' (errno: 24)

Ciò è causato dal fatto che esiste un limite (a livello di sistema) al numero di file che l'utente mysql può tenere aperti contemporaneamente. Da ubuntu 15.04 creare il file (anche la cartella mysql.service.d se serve)

/etc/systemd/system/mysql.service.d/override.conf

ed inserire le righe

[Service]
LimitNOFILE=5000

Now reload the Systemd configuration with:

 sudo systemctl daemon-reload

Per sistemi più vecchi modificare il file  /etc/security/limits.conf come segue

[...]
#@student        -       maxlogins       4
mysql       soft    nofile      4096
mysql       hard    nofile      8192

# End of file

Anche MySQL stesso ha limiti sul numero di file aperti e sul quantitativo di cache utilizzabile, modificare quindi i seguenti parametri (sempre nella sezione [mysqld]) per evitare eventuali problemi

table_cache     = 512
open-files-limit    = 2048
