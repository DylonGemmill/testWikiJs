In caso di ambienti con Zimbra, dove non è possibile installare Postfix in quanto già fornito dal groupware, sul sistema non è possibile inviare le mail tramite l'omonimo comando. In passato abbiamo usato sSMTP ( Installazione e configurazione di sSMTP), ma ha lo svantaggio di non gestire le code quindi se il server non è disponibile nel momento in cui si invia la mail questa viene persa. Un'alternativa molto migliore è nullmailer.

aptitude install nullmailer

Alle domande dell'installer, come nome del server di posta fornire il dominio mail del server, e come server remoto l'hostname locale.

Al termine aggiungere in ?/etc/nullmailer/remotes altri possibili relay, come ad esempio quello della linea a cui si è connessi (in ufficio usiamo mailout.briantel.com, per telecom può essere mail.cs.interbusiness.it). Si possono specificare più server smtp, uno per linea. Se non specificato niente altro vengono considerati server smtp sulla porta standard (25). E' possibile configurare anche relay con autenticazione, o cambiare la porta. Per tutte le possibili configurazioni, vedere la sezione remotes della man page di nullmailer-send.
