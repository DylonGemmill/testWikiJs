ISPConfig usa php-fcgi per gestire le pagine php in modo performante. L'installazione di default di ISPConfig ha una dimensione massima dei file caricabili molto bassa (131072). In caso di upload di file più grandi restituisce un errore 500, e nei log dovremmo trovare dei messaggi di questo tipo:

/var/www/xxx.it/log/error.log:
[Tue Feb 05 11:49:38 2013] [warn] [client 93.36.217.97] mod_fcgid: HTTP request length 131420 (so far) exceeds MaxRequestLen (131072), referer: http://xxx.it/wp-admin/media-new.php

 

il parametro si può modificare in due punti, per dominio o per server. Per dominio andare nelle impostazioni avanzate ed aggiungere:

FcgidMaxRequestLen 4000000

 

per modificarlo a livello di server aprire /etc/apache2/mods-available/fcgid.conf e modificare/aggiungere la riga:

MaxRequestLen 4000000

 

Oltre a questo va comunque configurato anche il parametro upload_max_filesize di php. Creare il file /etc/php5/conf.d/yetopen.ini con questo contenuto:

upload_max_filesize=4M
