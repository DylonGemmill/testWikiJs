Per elencare le mail di un account usare il comando

zmmailbox -z -m ACCOUNT s -t message -l 20 "in:inbox date:02/02/2018"

tra apici è possibile specificare una qualunque query di ricerca, come quella di esempio che cerca tutte le email del 2 febbraio 2018 nella posta in arrivo. -l  è il numero di risultati, il massimo è 1000 il predefinito 25.

Il risultato sarà qualcosa di simile:

num: 3, more: false

      Id  Type   From                  Subject                                             Date
   -----  ----   --------------------  --------------------------------------------------  --------------
1. 74313  mess   info                  Re: meeting                                         02/20/18 08:14
2. 73555  mess   Morsal                Re: meeting                                         02/19/18 14:02
3. 74315  mess   info                  Re: meeting                                         02/16/18 10:06

L'importante è la prima colonna, ovvero l'ID del messaggio. Per leggerlo:

zmmailbox -z -m ACCOUNT getMessage IDMESSAGGIO

verrà stampato a video con delle intestazioni minimali. È possibile vedere dati aggiuntivi di Zimbra passando --verbose prima dell'ID del messaggio, ma non è possibile vedere le intestazioni complete dell'email. 

Per vedere il messaggio completo su disco bisogna fare un giro più lungo: trovare prima l'id della mailbox e poi fare una query per ottenere il messaggio. 

Per cercare l'ID della casella:

zmprov getMailboxInfo ACCOUNT

Otterremo l'id e la dimensione occupata. Come prima cosa bisogna trovare il gruppo di mailbox dove è memorizzata la casella, lanciando in una shell bash

expr IDMAILBOX % 100

Prendere nota del risultato del comando sopra che ci fa ottenere il gruppo della mailbox ed entrare nel MySQL di zimbracon il comando:

mysql mboxgroupIDMAILBOXGROUP

Per trovare il messaggio dobbiamo fare una query su MySQL di Zimbra. La query concatena alcune stringhe con i dati presi dal DB. Una cosa a cui fare attenzione è la parte iniziale del percorso, perché si basa sul path predefinito dello store che è /opt/zimbra/store. Se fosse in uso un volume (ottenibile con il comando zmvolume -l) diverso occorrerà cambiarlo. Il comando da lanciare:

select id, 
    concat('/opt/zimbra/store/', (mailbox_id >> 12), '/', mailbox_id, '/msg/',
    (id >> 12), '/', id, '-', mod_content, '.msg') as file
    from mail_item where mailbox_id="MAILBOXID" and id="MESSAGEID" limit 1;

Il file ottenuto è il blob del messaggio su disco.
