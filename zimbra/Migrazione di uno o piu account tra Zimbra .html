zmztozmig è un tool di migrazione di Zimbra che consente di copiare una casella da un'installazione di Zimbra all'altra in modo automatico e comodo. Di fatto crea un tgz dell'account (come fa l'esportazione account, in preferenze) e lo carica sulla nuova installazione. Il vantaggio è che si può schedulare per più utenti e gira da linea di comando.

Può essere esguito indistintamente dalla macchina target o source, o in alternativa da un qualunque sistema dove sia installato Zimbra, l'importante è che abbia accesso all'amministrazione (porta 7071 https) di entrambi i sistemi.

 

Come prima cosa aprire il file /opt/zimbra/conf/zmztozmig.conf. I parametri fondamentali da modificare:

  • SourceZCSServer: host del server Zimbra sorgente
  • SourceAdminUser: utente admin della macchina di partenza (di solito admin)
  • SourceAdminPwd: password dell'utente
  • TargetZCSServer: host del server Zimbra di destinazione
  • TargetAdminUser: utente admin server di destinazione
  • TargetAdminPwd: password dell'utente
  • Threads: numero di processi contemporanei da eseguire. Considerare la banda occupata durante il trasferimento e che sul server di destinazione per ogni elemento caricato dovrà essere eseguita l'indicizzazione
  • DomainMap: se il dominio di partenza e di destinazione non coincidono occorre specificare qui la conversione. Ad esempio se il server di partenza è @yetopen.it e la destinazione dovrà essere @yetopen.com, aggiungere una riga 
    DomainMap=yetopen.it yetopen.com
  • Accounts: lista degli account (indirizzi email) da migrare, separati da virgola. Se si vuole importare tutti gli account usare la keyword all
  • Domains: in caso si usi Accounts=all occorre specificare l'elenco dei domini da migrare, dal server mittente a quello di destinazione. Va specificato come un elenco di domini separati da virgola
Quindi come utente zimbra eseguire il comando
/opt/zimbra/libexec/zmztozmig
