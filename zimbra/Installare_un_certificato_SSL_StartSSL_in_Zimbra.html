Creazione del certificato

Registrarsi e accedere a https://auth.startssl.com.

Effettuare la certificazione di proprietà di un dominio, andando in Validations Wizards > Domain name validation. Immettere il nome dominio (TLD) e proseguire. Verranno offerti alcuni indirizzi email con cui sarà possibile confermare il possesso del dominio. Potrebbe essere necessario creare un indirizzo email per la verifica. Normalmente facciamo creare un alis postmaster@dominio.it che inoltra a support@yetopen.it. Richiesto l'invio della mail arriverà a quell'indirizzo un codice di verifica, da fornire al wizard di validazione.

Validato il dominio potremo richiedere il certificato per il sito. Andare in Certificates Wizard > Web Server SSL/TSL Certificate. Se non si è preparata una CSR, immettere una password di almeno 10 caratteri (e memorizzarla!), e proseguire. Copiare il contenuto della textbox ed incollarlo in un file sul server con li nome ssl.key, facendo attenzione a non copiare spazi o creare nuove linee non presenti nel riquadro originale. Decriptare la chiave con il comando seguente, usando la password creata prima:

openssl rsa -in ssl.key -out ssl.key

Proseguire sul pannello web StartSSL e selezionare il dominio per cui si sta richiedendo il certificato. Proseguire ed inserire l'hostname completo dell'host da certificare (terzo livello), che è poi l'indirizzo del sito web a cui si accede con Zimbra.

Una volta che il certificato viene rilasciato, scaricarlo come ssl.crt sul server. Importante! Verificare che nell'ultima riga del file ci sia un newline, ovvero una riga vuota dopo -- END CERTIFICATE--, altrimenti il deploy in Zimbra 7 potrebbe fallire!

Installazione del certificato

Copiare ssl.crt e ssl.key in /tmp/.

cd /tmp
wget http://www.startssl.com/certs/ca.pem
wget http://www.startssl.com/certs/sub.class1.server.ca.pem
cat ca.pem sub.class1.server.ca.pem > ca_bundle.crt
cp ssl.key /opt/zimbra/ssl/zimbra/commercial/commercial.key 
cd /opt/zimbra/bin

Deployamo il certificato:
./zmcertmgr deploycrt comm /tmp/ssl.crt /tmp/ca_bundle.crt

E poi lanciamo:
/opt/zimbra/java/bin/keytool -import -alias new -keystore /opt/zimbra/java/jre/lib/security/cacerts -storepass changeit -file /opt/zimbra/ssl/zimbra/commercial/commercial.crt


su - zimbra
zmcontrol restart

ref http://wiki.zimbra.com

 In caso si rinscontrassero problemi con l'esistenza dell' alias procedere all'eliminazione con questo comando:

keytool -delete -alias my_ca -keystore /opt/zimbra/java/jre/lib/security/cacerts -storepass changeit

Per apache2:

 

SSLCertificateFile /dati/cert/mail.ufficyo.com/2012/ssl.crt
SSLCertificateKeyFile /dati/cert/mail.ufficyo.com/2012/ssl.key
SSLCACertificateFile /dati/cert/mail.ufficyo.com/2012/ca_bundle.crt
