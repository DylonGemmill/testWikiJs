In alcuni casi (da capire, forse quando c'è abilitato sia http che https) gli elementi della valigetta non si aprono correttamente, ad esempio usando un url http invece che https.

Per correggere questa situazione occorre impostare tre parametri con zmprov. Verificare cosa occorre modificare con il comando:

zmprov gad -v |grep zimbraPublicService

I tre parametri che servono sono 

  • zimbraPublicServiceHostname: nome host (dns)
  • zimbraPublicServiceProtocol: deve essere http o https 
  • zimbraPublicServicePort: tipicamente 80 per http e 443 per https

Se uno di questi manca impostarlo con questi comandi

zmprov md <domain> zimbraPublicServiceHostname mail.dominio.it
zmprov md <domain> zimbraPublicServiceProtocol https
zmprov md <domain> zimbraPublicServicePort 443
