Migrazione posta

Il metodo più semplice, ove possibile, è usare IMAP.

 

ImapSync

Con ImapSync è possibile copiare direttamente dal server di partenza a Zimbra.

ImapSync è in grado di copiare le email tenendo traccia dei duplicati, ovvero è possibile eseguirlo più volte senza problemi.

Per installare IS:

git clone https://github.com/imapsync/imapsync.git /dati/yetopen/imapsync 
cd /dati/yetopen/imapsync 
apt-get install libmail-imapclient-perl libio-socket-inet6-perl libfile-tee-perl libunicode-string-perl libreadonly-perl libsys-meminfo-perl libregexp-common-perl libfile-tail-perl

non va compilato, è possibile eseguirlo direttamente dalla directory dove è stato eseguito il clone.

 

L'ideale per la migrazione della posta è prima fare un passaggio a caldo che porti il grosso delle email, quindi fermare lo scaricamento sul vecchio server e fare un nuovo sync a freddo, in modo da prelevare velocemente le ultime email scaricate. Ad operazione terminata si può far partire lo scaricamento sul nuovo server.

I parametri ottimali per il primo passaggio di imapsync sono:

--host1 OLDSERVER --port1 143 --host2 NEWSERVER --port2 143 --exclude user --syncinternaldates  --nofoldersizes --skipsize --fast --addheader --delete2

attenzione al parametro --delete2: questo fa si che le mail presenti sul server di destinazione ma NON su quello di partenza vengano cancellate. In questo modo se l'utente ha lavorato sul primo server cancellando delle email queste verranno cancellate anche sul secondo. 

La --exclude invece esclude dalla migrazione tutte le cartelle di posta (e figlie) che iniziano con quel nome. In questo caso è utile per i mailserver cyrus, dove le mailbox condivise erano accessibili nelle sottocartelle user/.

Oltre a queste opzioni vanno aggiunte quelle per l'autenticazione:

--user1 UTENTE_HOST1 --password1 PASSWORD_HOST1 --user2 UTENTE_HOST2 --password2 PASSWORD_HOST2

Gmail

Nelle versioni più recenti Imapsync ha un'opzione --gmail1 (o 2, in caso si voglia migrare verso) per indicare un server Gmail. Selezionando questo parametro si possono omettere host e port e gli altri parametri opzionali del server.

Tra le altre include le opzioni per gestire la cartella Tutti i messaggi in modo che non duplichi email già migrate su altre cartelle (etichette). Nel caso della localizzazione italiana va aggiunto in fondo al comando il parametro:

--folderlast "[Gmail]/Tutti i messaggi"

Vedere qui per tutte le info.

Sincronizzazione in una sottocartella

Se vogliamo copiare l'account1 in una sottocartella di account2 possiamo usare l'opzione --subfolder2 CARTELLA_DI_DESTINAZIONE. In questo modo tutte le cartelle della sorgente verranno create sotto la cartella indicata sulla destinazione.

 

Thunderbird

Nel qual caso l'utente abbia la posta su un client come thunderbird creare un nuovo account IMAP verso il server Zimbra e copiare tutte le cartelle usando il drag&drop dalle cartelle locali a quelle di Zimbra. Lo svantaggio rispetto ad imapsync è che tiene impegnato il client.

 

Zimbra migration tool

Questo strumento ha funzionalità simili a quelle di imapsync ma risulta meno versatile. Conviene spesso usare la prima soluzione.

 

Migrazione contatti

Da Thunderbird era possibile usare Zindus, ma non è più mantenuta e non funziona con le versioni recenti di Thunderbird.

E' possibile però esportare i contatti come CSV e convertirli in un formato digeribile a Zimbra con uno script python presente sul forum Zimbra (e in repo yo)
