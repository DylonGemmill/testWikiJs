Eseguendo un upgrade di ram su un sistema a 32bit, è possibile che alcuni servizi di Zimbra, in particolare mailboxd, non partano. Verificando in /opt/zimbra/log/zmmailboxd.out ho trovato il seguente errore:

Error occurred during initialization of VM
Could not reserve enough space for object heap
Could not create the Java virtual machine.

Questo perché la Java VM a 32 bit non poteva allocare la memoria aggiuntiva.

Per risolvere:

su - zimbra
zmlocalconfig -e mailboxd_java_heap_memory_percent=18
zmcontrol restart
