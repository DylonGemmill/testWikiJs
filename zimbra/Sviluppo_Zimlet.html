Inserire una voce di menu nell'overview

Di seguito un esempio di codice da inserire nel file XML di definizione

  <zimletPanelItem label="Docucross" icon="Knowledgetree-panelIcon">
    <toolTipText>
                 Drag n' Drop per salvare un'email in Docucross
    </toolTipText>
    <dragSource type="ZmMailMsg" />
    <dragSource type="ZmConv" />
    <contextMenu>
            <menuItem label="Preferences" id="PREFERENCES" icon="KTPreferences"/>
            <menuItem label="" id=""/>
            <menuItem label="KT DMS"  id="WEB" icon="KTWebsearch"/>
    </contextMenu>

  </zimletPanelItem>

Il contextMenu è il menu che compare col click destro sull'icona.

 

Impostare Zimbra per ricaricare i file JSP ad ogni interrogazione

In caso di sviluppo di Zimlet che fanno uso della parte server JSP, è utile impostare Zimbra in modo che ricompili i file ad ogni interrogazione, altrimenti occorrerebbe un zmmailboxdctl restart dopo ogni modifica ad un file JSP.

Per vare ciò occorre aprire il file /opt/zimbra/jetty/etc/webdefault.xml e modificare i valori di development, checkInterval e reload-interval da così

    <init-param>
        <param-name>development</param-name>
        <param-value>false</param-value>
    </init-param>
    <init-param>
        <param-name>checkInterval</param-name>
        <param-value>0</param-value>
    </init-param>
    <init-param>
        <param-name>reload-interval</param-name>
        <param-value>-1</param-value>
    </init-param>

a così

    <init-param>
        <param-name>development</param-name>
        <param-value>true</param-value>
    </init-param>
    <init-param>
        <param-name>checkInterval</param-name>
        <param-value>1</param-value>
    </init-param>
    <init-param>
        <param-name>reload-interval</param-name>
        <param-value>1</param-value>
    </init-param>

Riavviare quindi jetty col comando

$ zmmailboxdctl restart

da utente zimbra

 

Se dovessero esserci errori con i file JSP, jetty mostra l'errore generico 500. Per ulteriori dettagli consultare il file di log /opt/zimbra/log/zmmailboxd.out

 

Utilizzare i messaggi di sistema di Zimbra

Per visualizzare un messaggio di sistema per mezzo delle librerie di Zimbra utilizzare questa chiamata

appCtxt.getAppController().setStatusMsg(msg, ZmStatusView.LEVEL_INFO);

 

dove msg è una stringa contenente il messaggio da visualizzare ed i livelli di messaggio a disposizione sono:

  • CRITICAL - il messaggio è accompagnato da una X rossa, solitamente identifica un errore
  • WARNING - il messaggio è accompagnato da un'icona con il triangolo giallo di pericolo, serve per richiamare l'attenzione
  • INFO - il messaggio è accompagnato da una V verde, spesso identifica il successo di un'azione
