Attenzione! Usare filebeat 6.7 perché la versione 7.x usa un logging diverso e non è (per ora) compatibile con logz.io. 

mkdir -p /etc/pki/tls/certs
wget https://raw.githubusercontent.com/logzio/public-certificates/master/COMODORSADomainValidationSecureServerCA.crt -P /etc/pki/tls/certs
wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | apt-key add -
apt -y install apt-transport-https
echo "deb https://artifacts.elastic.co/packages/6.x/apt stable main" | sudo tee -a /etc/apt/sources.list.d/elastic-6.x.list
apt update && apt install filebeat

Configurare /etc/filebeat/filebeat.yml. Ci sono dei file di esempio in NC/Sicuritalia/logz.io, altrimenti bisogna usare il wizard (richiede login).

Importante usare il token di SI (FmHZbOZVYAdnVcGnaYoQTvgNtVSKpWAP).

Avviare ed abilitare filebeat al boot con:

systemctl start filebeat ; systemctl enable filebeat

Verificare l'avvio con 

systemctl status filebeat

oppure in /var/log/syslog e /var/log/filebeat/filebeat.
