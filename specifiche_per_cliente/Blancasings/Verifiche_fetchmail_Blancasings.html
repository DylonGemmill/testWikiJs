fetchmail viene eseguito ogni 5 minuti tramite il crontab dell'utente root (crontab -l).

Ad ogni esecuzione viene salvato il log in /var/log/fetchmail.log. Purtroppo non contiene data ed ora di tutte le operazioni, quindi per vedere se il log è stato scritto di recente (e quindi se fetchmail è stato eseguito) facciamo un

ls -la /var/log/fetchmail.log
-rw-r--r-- 1 root root 10059305481 21 gen 09:30 /var/log/fetchmail.log

Se data ed ora sono negli ultimi 5 minuti allora fetchmail è stato eseguito.

Per capire se l'ultima esecuzione è andata a buon fine in fondo al log dovrebbe sempre esserci un messaggio come questo:

fetchmail: normal termination, status 0

L'importante è vedere normal termination, il codice di stato può essere 0 o 1.

Per vedere le ultime 10 righe del file usiamo il comando:

tail /var/log/fetchmail.log

Possibili errori

Se c'è questo messaggio:

fetchmail: another foreground fetchmail is running at 16251.
fetchmail: terminated with signal 15

significa che fetchmail è stato eseguito ma c'era già un'altra istanza attiva. Se il messaggio compare una o due volte, sebbene strano può essere normale. Se continua a ripetersi probabilmente va terminato il processo indicato nel log, in questo caso 16251.
