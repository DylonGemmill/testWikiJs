BACKUP SINGOLI CLIENT DEI COLLABORATORI

Script di backup che memorizza i seguenti dati sulla home del collaboratore creata sul server:

  • /home/collaboratore/Documenti (cfr directory standard ubuntu)

  • /home/collaboratore/.gpg

  • /home/collaboratore/.ssh

  • /home/collaboratore/.vim

  • /etc/openvpn

Dettagli dello script: Script di backup PC operatori

 

BACKUP SUL SERVER

  • cestino di rete

  • rdiff-backup con strico di 15gg

BACKUP LOCALE

A) Viene eseguito 6 volte al giorno (alle 24.00, 10.00, 13.00, 16.00, 19.00 e 21.00) e memorizzato su disco USB in /mnt/usb/backup-quaglia. Vengono salvati i seguenti dati:

  • cartella /home

  • dump di mysql

  • /var/www + /opt/otrs (spostate nella /home )

  • macchina asterisk

B) Viene eseguito 4 volte al giorno (alle 07.00, 13.00, 21.00 e 24.00) e fa un backup di zimbra a caldo prima e a freddo dopo, su disco USB.

BACKUP REMOTO

Viene eseguito 3 volte al giorno (alle 06.00, 21.00 e 24.00) ed effettua un rsync verso un server xen remoto (altro edificio)

BACKUP FIREWALL

Una volta a settimana il firewall invia una mail con un backup della configurazione a support@yetopen.it.
