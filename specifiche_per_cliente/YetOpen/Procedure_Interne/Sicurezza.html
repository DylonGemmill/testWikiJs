Questo perche’ se memorizzate in modo poco sicuro, si perde l’efficacia di una password forte.

DATI CLIENTI (NC, Zimbra,ecc...):

  • tutti i dati dei clienti dovranno passare su BitWarden (Lorenzo sta verificando il discorso backup)
  • devono essere attivate le funzioni di Zimbra e NC di remote wipe (e vanno anche testate nel dettaglio)
  • dobbiamo configurare TUTTI gpg per criptare le email. In caso di informazioni sensibili che riguardano i clienti, le email vanno criptate
  • per quanto riguarda il discorso di criptare i dati in NC sul server (in modo che se bucano il server di Contabo i dati non sono leggibili) dobbiamo verificare se si puo’ fare
  • nel caso di cartelle particolarmente sensibili salvate su NC e nel caso non si possa criptare tutto, si puo’ criptare la sola cartella che, a quel punto, risultera’ non leggibile fino a che non verra’ montata sul client (gocryptfs)
  • nel caso di file particolarmente sensibili non inseribili in BitWarden, si puo’ usare la criptazione simmetrica di gpg (--symmetric). In questo modo il file e’ decriptabile con il solo inserimento di una frase che viene richiesta in fase di criptazione (non e’ la propria passphrase!)
  • Creare script per ricercare file degli appunti che non sono criptati, se ce ne sono che non devono essere criptati, vanno rinominati. Oppure rinominiamo i file in *_secure_* cosi’ abbiamo la discriminante (cerca con grep password o ip o accesso o altro)

NOTEBOOK AZIENDALI:

  • all’accensione deve esserci password per avvio e per accesso al BIOS
  • attivare filesystem criptato o quantomeno la cartella HOME (ci sono i dati di NC - rif. https://www.howtogeek.com/116032/how-to-encrypt-your-home-folder-after-installing-ubuntu/)
  • il blocco schermo deve essere protetto da password e si deve attivare automaticamente dopo 5 minuti di inutilizzo

CELLULARI AZIENDALI:

  • All’accensione deve esserci password per avvio
  • il blocco schermo deve essere protetto da password e si deve attivare automaticamente dopo 5 minuti o allo spegnimento del display

N.B. dovremo avere file criptato su NC a disposizione degli amministratori di sistema che contenga le password utili al ripristino del notebook o del cellulare. ATTENZIONE: non si fa riferimento alle password degli account personali ma alle password che consentano un eventuale ripristino dei dispositivi

N.B.2 ATTENZIONE ai backup o alle copie dei file e dei documenti aziendali… se vengono lasciate “in giro” e’ un grosso problema di sicurezza

BIGINO COMANDI GPG (per il dettaglio si veda il doc PGPmanual.pdf) :

gpg --gen-key
gpg --output revoca.asc --ger-revoke mia_chiave
gpg --list-keys
gpg --output pippo.gpg --export pippo@pluto.org -> esporta chiave pubblica
gpg --armor --export pippo@pluto.it -> esporta con ASCII armor
gpg --import pippo.gpg
gpg --edit-key pippo@pluto.it
comando> frp -> visualizza fingerprint
comando> sign -> firma la chiave
comando> check -> verifica integrita' della chiave e visualizza chi ha firmato
gpg -e -> cripta
gpg -d -> decripta
gpg --ouput doc.gpg --symmetric doc -> cripta in modo simmetrico (con passphrase)
gpg --output doc.sign --sign doc -> firma e cripta un documento
gpg --verify doc.sign -> verifica la firma del documento
gpg --decrypt doc.sign -> verifica e decripta il documento
gpg --clearsign doc -> firma solamente il file (il file resta in chiaro)
gpg --detach-sig doc -> firma il documento inserendo a firma in un nuovo documento (l'originale resta immutato)
gpg --verify doc.sig doc -> per verificare un documento firmato con firma staccata
comando> toggle -> switcha tra chiave pubblica e privata (se sono disponibili entrambe)
comando> adduid -> aggiunge un nuovo user id
comando> addkey -> aggiunge una chiavve
comando> key 2 -> seleziona la chiave 2
comando> uid -> seleziona lo userid
comando> deluid -> elimina lo userid
comando> delkey -> elimina la chiave
comando> revkey -> revoca la chiave selezionata
