Il file degli appunti è un file criptato presente nella directory di ogni cliente con tutte le informazioni sulla rete, sui server, gli apparati, gli account personali e in generale tutto quello che, di riservato, riguarda quel cliente.

Oltre agli appunti del cliente c'è un file degli appunti di YetOpen che si chiama info_rete.txt.gpg: in questo file devono essere memorizzate le credenziali dei siti di uso generico all'interno dell'ufficio.

Cosa inserire negli appunti di un cliente

Tutto ciò che è specifico del cliente, quindi:

  • password di account o altri dati delicati: se ad esempio dobbiamo avere accesso al portale di supporto di un prodotto (zimbra), oppure un account per il ticketing di un fornitore, servizi di hosting...
    Attenzione, perché soprattutto con il GDPR, è meglio non avere accesso a caselle di posta, carte di credito, servizi di pagamento e simili
  • password di computer, account locali alla rete, accessi database
  • chiavi API di servizi esterni (Google Maps, Google Analytics...)
  • server (fisici e virtuali)
  • indirizzi IP, privati e pubblici
  • chiavi di cifratura, informazioni di backup

Cosa memorizzare in info_rete

Tutto ciò che è di pertinenza YetOpen e tendenzialmente è meglio non memorizzare in chiaro.

Stile di inserimento

Ogni server/servizio deve iniziare con un titolo racchiuso da asterischi. Il titolo deve rappresentare il nome del servizio oppure il nome del server oppure il prodotto...

Di seguito vanno messe le informazioni di dettaglio, come ad esempio url di accesso, password. In generale teniamo la sintassi
campo:<1 tab>valore
Se ci sono più campi da inserire e di lunghezza diversa che andrebbero a disallineare i valori si possono aumentare i tab in modo da tenere tutti i valori incolonnati allo stesso livello.

Se il blocco è diventato obsoleto perché sostituito da altro oppure dismesso inserire sotto il titolo, separato da una riga vuota, il testo +++ OBSOLETO +++. Se necessario aggiungere una riga di spiegazioni e ripetere sotto lo stesso OBSOLETO sopra (come fosse un blocco a sé).
Spostare tutto il blocco obsoleto in fondo al file.

Alcuni esempi:

Credenziali di accesso ad un sito

**********
* Sito A *
**********
http://url_del_sito/
utente:    yetopen
password:  y3t0p3n

Invece che fare una riga per utente e una per password siccome al 99% c'è sempre una coppia di questo tipo si possono indicare come:

**********
* Sito A *
**********
http://url_del_sito/
<1 tab>yetopen:<1 tab>y3t0p3n
<1 tab>utenteb:<1 tab>passwordb

In questo modo si possono indicare più credenziali in fila, indentando gli utenti di un tab così da far capire che sono relative al servizio sopra.

Dispositivo hardware (NAS, server)

Nel caso di dispositivi hardware è utile segnare anche marca, modello, numero di serie ed indirizzi mac, in caso sia necessario cercarli sulla rete (o sul DHCP) oppure registrarli per la garanzia.

Ad esempio per memorizzare una webcam:

**********************
* WebCam casa Pietro *
**********************
Avtech AVM2421AP/F28
S/N:<2 tab>      ABC123456
IP: <2 tab>      DHCP
Mac:<2 tab>      00:0a:f3:ss:bb:cc
Porta web:<2 tab>88

admin:<2 tab>    Passw0rd

Anche se sono abbastanza distinguibili vale la pena separare le proprietà dell'apparato dagli utenti di accesso. In generale dove c'è admin si presume che sia l'utente di accesso.

Nel caso ci siano da segnare dei sottoservizi di un elemento, come un elenco, come fatto sopra si indenta di un tab. Nel caso di un modem con dei port forward:

*******************
* ADSL Casa Tizio *
*******************
Linea Vodafone

UPnP Disabilitato

Port Forward / NAT:
<1 tab>8080 -> NAS
<1 tab>8888 -> Webcam

Server con servizi

Come indicato sopra in questo caso avremo diversi sottoservizi da indentare. È utile indicare, ad esempio per servizi come MySQL o LAM, l'url diretto di accesso.
Per i server Linux è anche importante indicare il nome host di accesso per yossh.

********************
* Server GestYOcar *
********************
IPv4:<1 tab> 1.2.3.4
IPv6:<1 tab> 2a00:a:b:c::1/64
root:<1 tab> abcDEF123   (password originale DEFabc543)
yossh:<1 tab>nome_host_yossh

VNC console:<1 tab>4.3.2.1:63056 password Passw0rd

MySQL: https://host.name.it/__Yo_Adminer.php
<1 tab>root:<3 tab>    Passw0rd
<1 tab>backup-user:    Passw1rd
<1 tab>gestyocar:      Passw2rd

Backup con duply+duplicity su Amazon S3 (YetOpen) bucket yetopen-backup-esempio
Chiave di codifica duplicity: chiave-segreta=

Utenti applicativo GestYOcar: https://host.name.it/
<1 tab>yetopen:<1 tab>Passw4rd

In questo caso è un server virtuale su Contabo, quindi ha anche la console VNC di accesso in caso di ripristino.

In ogni caso separare i diversi servizi all'interno dello stesso blocco (server, ad esempio) con una riga vuota prima e dopo.
