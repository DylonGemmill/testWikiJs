Per effettuare l'accesso alle cartelle contenenti il backup è possibile scaricare WINSCP  al seguente link:

https://winscp.net/eng/download.php 

(eventualmente è anche possibile utilizzare filezilla https://filezilla-project.org/  )

Cliccare su "Installation package" scaricare l'eseguibile ed installarlo.

Una volta avviato WINSCP comparirà una finistra come quella seguente:

Compilare i campi richiesti con le seguenti informazioni:

Protocollo file: SCP

Nome server: 192.168.1.99

Nome Utente: root

Password: QUELLA DA NOI FORNITA

E cliccare su Accedi.

A questo punto, bisogna accedere al percorso /dati/backup sfogliando le cartelle come da immagini:

Arrivati al contenuto della nostra cartella di backup, vedremo che sono presenti diverse cartelle:

- Due cartelle hourly.x, la hourly.0 che contiene il backup orario più di recente della data attuale e hourly.1 che contiene quello più vecchio.

I backup vengono effettuati alle ore 13:00 e alle ore 22:00, perciò se osserviamo i backup alle ore 14:00 la directory hourly.0 conterrà il backup delle 13:00 e la directory hourly.1 conterrà il backup delle 22:00 del giorno prima.

- Sette cartelle daily.x dove sono presenti i backup giornalieri svolti ogni giorno della settimana alle ore 21:28.

Anche in questo caso il numero più basso di daily.x risale a quello più recente e così sarà per tutti gli altri tipi di backup.

Quindi in questo caso se andassimo a vedere lo stato dei backup alle ore 23:00, avremo il backup daily.0 che contiene il backup di tutta la giornata e il backup dailiy.1 che contiente il backup del giorno prima, fino ad arrivare al daily.6 che contiente il backup di 7 giorni fa.

Il meccanismo spiegato più volte è identico anche per i backup weekly.x e monthly.x . 

-Quattro backup weekly.x dove sono presenti i backup svolti una volta la settimana di domenica alle 20:50.

-Tre cartelle monthly.x in cui sono presenti i backup effettuati ogni primo giorno del mese, quest'ultimi vengono mantenuti per 3 mesi.

Dunque a seconda del periodo temporale in cui il file e stato cancellato o danneggiato, bisogna accedere alla cartella (hourly.x, daily.x, ecc..) che più si avvicina al primo backup utile in cui si pensa che il file sia stato compromesso.

All'interno di ogni cartella appena citata troveremo poi altre due cartelle divise per tipo di backup la cartella srvhome e la cartella windowsshare.

Accedere alla directory srvhome per rispristinare i file delle condivisioni della macchina 192.168.1.198, invece accedere alla directory windowshare per rispristinare i file 192.168.1.16.

Una volta identificato il file corretto da ripristinare è possibile procedere nella copia dello stesso sulla cartella locale di destinazione.

Quindi sulla finestra di sinistra di WinSCP selezioneremo la cartella di destinazione, dove dovrà essere rispristinato il file.

A questo punto l'ultima operazione da effettuare sarà quella di trascinare il file di backup spostando il file dalla finestra di destra del backup remoto alla finestra di sinistra.
