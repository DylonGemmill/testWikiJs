E' possibile impostare una relazione 1 a 1 tra un IP pubblico ed un IP della LAN, in modo da evitare di dover effettuare il forward per ogni porta necessaria.

 

  1. configurare un IP in Firewall > Virtual IP di tipo Proxy ARP, con l'IP pubblico da utilizzare;
  2. andare in Firewall > NAT > 1:1 ed aggiungere una regola con questi parametri:
    Interface: WAN
    External subnet IP: IP pubblico
    Internal IP: Single Host 10.22.22.x (IP privato a cui inoltrare le porte) 
  3. Firewall > Rules > WAN e premere + per creare una nuova regola:
    Action: Pass
    Interface: WAN
    Protocol: any
    Destination: Single host or alias 10.22.22.x (IP privato) 
Dopo queste operazioni qualunque porta dell'IP pubblico dovrebbe essere direzionata a quello privato, e in uscita l'host dovrebbe avere l'IP pubblico specificato sopra.
